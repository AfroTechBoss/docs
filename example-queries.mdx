## Basic Price Queries

The most common use case — checking a live stablecoin price without leaving your editor or AI assistant.

### Single Asset Price

**Prompt:**

> "What is the current USDT/USD price from IFÁ Labs on Base Mainnet?"

**Tools called:** `get_asset_price`

**Response:**

```text
The current USDT/USD price on Base Mainnet is $1.000124.
The feed was last updated 412 seconds ago.
```

## Tips for Writing Effective Prompts

Getting the best results from the IFÁ Labs MCP Server comes down to specificity. Here's what works:

**Always specify the asset in `"SYMBOL/USD"` format when possible.** "CNGN price" works, but "CNGN/USD price on Base Mainnet" is unambiguous and gives the model everything it needs to call the right tool with the right parameters without guessing.

**Specify the network when it matters.** If you're working on a testnet integration, say "Base Sepolia" explicitly. Without a network qualifier, the server defaults to Base Mainnet.

**State your threshold when checking freshness.** "Is the feed fresh?" leaves the threshold ambiguous. "Is the feed fresh within a 30-minute window?" gives the tool a precise `max_age_seconds` value to evaluate against.

**Chain multiple requests in a single prompt for complex workflows.** The MCP client will execute multiple tool calls in sequence. "Get the CNGN price, check its freshness, and give me its asset ID" triggers three tool calls in one prompt rather than requiring three separate messages.
